# EverMemory Docs README

## 1. 文档目标

本目录已收敛为 `0.0.1` 发布与后续运维的统一文档入口，避免历史文档分散导致执行口径不一致。

---

## 2. 先读这 6 份（发布关键）

1. `evermemory-docs-index.md`（全量索引与分层）
2. `evermemory-branch-and-release-governance.md`（分支治理与工作树纪律）
3. `evermemory-release-quality-checklist.md`（发布门禁清单）
4. `evermemory-release-0.0.1.md`（本次正式版发布记录）
5. `evermemory-operator-runbook.md`（运行手册）
6. `evermemory-troubleshooting.md`（故障排查）

---

## 3. 文档分层（已整合）

- `L0 发布关键`：发布直接依赖，必须与代码同步更新
- `L1 运维操作`：runbook、troubleshooting、安装与验收
- `L2 设计与规划`：技术设计、路线图、阶段方案
- `L3 历史证据`：带日期的执行报告、审计报告、阶段性看板

具体文件归类见 `evermemory-docs-index.md`。

---

## 4. 更新纪律

1. 涉及发布流程变更，必须同步更新 `L0` 文档。
2. 涉及运行命令变更，必须同步更新 runbook 和 troubleshooting。
3. 新增日期类报告（`*-YYYY-MM-DD.md`）归入 `L3`，不得替代 `L0/L1` 标准文档。

---
